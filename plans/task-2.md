# Task 2: The Documentation Agent - Implementation Plan

## Архитектура агента с инструментами
1. **Инструменты (tools)**:
   - `read_file(path)`: читает содержимое файла
   - `list_files(path)`: показывает список файлов в директории

2. **Agentic loop**:
   - Отправляем вопрос + описания инструментов в LLM
   - Если LLM возвращает `tool_calls` → выполняем инструменты, добавляем результаты в историю, повторяем
   - Если LLM возвращает текст → это финальный ответ
   - Максимум 10 итераций (предотвращение бесконечных циклов)

3. **Безопасность**:
   - Проверка path: запрещаем `..` и абсолютные пути
   - Работаем только внутри проекта

4. **Структура ответа**:
   - `answer`: финальный ответ
   - `source`: ссылка на wiki (файл#секция)
   - `tool_calls`: история всех вызовов

5. **Системный промпт**:
   - Инструктируем использовать list_files для навигации по wiki
   - Затем read_file для чтения нужных файлов
   - Просим указывать источник в формате wiki/filename.md#section
